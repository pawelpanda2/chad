========================================
OPIS DZIAŁANIA
========================================

Ten skrypt służy WYŁĄCZNIE do uruchomienia istniejącego obrazu Docker.

1. Odczytuje lub znajduje ostatni IMAGE_TAG
2. Sprawdza czy image istnieje (docker image inspect)
3. Jeśli image nie istnieje, przerywa z błędem
4. **Najpierw wywołuje stop_docker_image.sh aby zatrzymać stare kontenery**
5. Uruchamia kontenery używając docker compose up --no-build
6. Używa portów testowych: 12020 (frontend), 12024 (Content Provider API)

========================================
PORTY
========================================

Frontend: 12020
Backend API: 12023
Content Provider API: 12024

========================================
KOMENDY
========================================

bash 03_scripts/nodejs/05_docker_qnap_test/start_docker_image.sh

bash 03_scripts/nodejs/05_docker_qnap_test/start_docker_image.sh --tag 26-06-10__18-00-00

# Lista dostępnych tagów
bash 03_scripts/nodejs/05_docker_qnap_test/start_docker_image.sh --list-tags

========================================
WAŻNE
========================================

- Skrypt używa --no-build przy docker compose up
- Jeśli image nie istnieje, skrypt przerywa z błędem
- Build musi być wykonany wcześniej przez build_docker_image.sh
- Skrypt nie generuje nowego taga samodzielnie
