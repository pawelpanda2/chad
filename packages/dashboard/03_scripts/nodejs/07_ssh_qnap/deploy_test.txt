Target env: .env (na QNAP)
Główne zadanie: Deploy aplikacji na QNAP test environment przez SSH
Czego NIE robi: Nie deployuje na produkcję, nie używa istniejących obrazów

========================================
OPIS DZIAŁANIA
========================================

deploy_test.sh jest wrapperem, który wywołuje deploy_env test z
server_by_ssh_v3.sh.

Flow:
1. Łączy się przez SSH z QNAP (konfiguracja z lokalnego .env)
2. git pull --ff-only w repozytorium na QNAP
3. Wywołuje copy-env.sh qnap_test na QNAP (przygotowuje .env)
4. cd 03_scripts/nodejs/05_docker_qnap_test
5. build_docker_image.sh (BUDUJE nowy obraz Docker)
6. stop_docker_image.sh (zatrzymuje stare kontenery)
7. start_docker_image.sh (uruchamia nowe kontenery)
