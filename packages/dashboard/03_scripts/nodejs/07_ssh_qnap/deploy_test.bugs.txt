Nazwa błędu: copy-env.sh: local_mac: unbound variable

Data: 2026-06-10

========================================
OBJAW
========================================

Podczas deploy na QNAP:

Running: bash ./copy-env.sh qnap_test
./copy-env.sh: line 35: local_mac: unbound variable

========================================
DIAGNOZA
========================================

To NIE był problem ze ścieżką ani SSH.
Skrypt się uruchamiał poprawnie, ale wywalał się wewnątrz `copy-env.sh`.

Przyczyna:
W `copy-env.sh` użyto wadliwej tablicy asocjacyjnej bash:

  declare -A ENV_MAP=(
    ["local_mac"]="02_local_mac"
    ...
  )

Przy `set -euo pipefail` odwołanie do nieistniejącego klucza w tablicy
asocjacyjnej powoduje błąd "unbound variable".

Mimo że wywoływano `copy-env.sh qnap_test`, błąd wskazywał na `local_mac`
ponieważ implementacja najpierw sprawdzała czy klucz istnieje w tablicy
poprzez `${ENV_MAP[${env_name}]+isset}` co przy `set -u` failowało.

========================================
POPRAWKA
========================================

Zastąpiono wadliwą tablicę asocjacyjną przez `case` statement:

  case "${ENV_NAME}" in
    local_mac)
      SOURCE_ENV_FILENAME=".env.02_local_mac"
      ;;
    docker_mac)
      SOURCE_ENV_FILENAME=".env.04_docker_mac"
      ;;
    qnap_test)
      SOURCE_ENV_FILENAME=".env.05_docker_qnap_test"
      ;;
    qnap_prod)
      SOURCE_ENV_FILENAME=".env.06_docker_qnap_prod"
      ;;
    *)
      echo "❌ Unknown env name: ${ENV_NAME}"
      exit 1
      ;;
  esac

========================================
WAŻNE
========================================

1. Mapa env name -> env filename została zachowana
2. `copy-env.sh` nie interpretuje zawartości plików env
3. Skrypt tylko kopiuje plik źródłowy do .env
4. Implementacja przez `case` jest bezpieczna przy `set -euo pipefail`

Oba powinny działać bez błędów.
