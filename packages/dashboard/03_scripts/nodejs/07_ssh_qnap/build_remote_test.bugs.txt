# build_remote_test.sh - Bugs & Issues

## Bug: Missing copy-env.sh call at script start

**Date:** 26-06-10

**Severity:** HIGH

**Description:**
Skrypt `build_remote_test.sh` nie wywoływał `copy-env.sh` na początku, co powodowało, że zmienne środowiskowe nie były poprawnie załadowane przed uruchomieniem skryptu.

**Impact:**
- `CONTENT_PROVIDER_API_URL` nie było ustawione (pokazywało "not set")
- Inne zmienne środowiskowe z `.env.05_docker_qnap_test` nie były dostępne
- Strona `/dashboard/folders` pokazywała błąd "Content Provider API unavailable" z debug info pokazującym "not set" dla API URL

**Root Cause:**
Skrypt od razu source'ował `server_by_ssh_v3.sh`, który korzysta ze zmiennych środowiskowych, ale nie wywoływał wcześniej `copy-env.sh` aby załadować odpowiedni plik `.env`.

**Fix:**
Dodano wywołanie `copy-env.sh` na samym początku skryptu, przed source'owaniem helpera SSH:

```bash
# Step 0: Prepare environment variables
# Must be done FIRST to ensure correct env is loaded
echo "[INFO] Preparing environment for qnap_test..."
bash "${SCRIPT_DIR}/../01_envs/copy-env.sh" "qnap_test"
```

**Verification:**
Po fixie, przed uruchomieniem skryptu należy zobaczyć:
```
[INFO] Preparing environment for qnap_test...
[INFO] copy-env.sh diagnostic
[INFO] Script path: ...
[INFO] Copied:
[INFO]   .env.05_docker_qnap_test
[INFO] -> .env
[INFO] Done.
```
